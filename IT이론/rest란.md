1. REST 구성
    * 자원(Resource)
    * 행위(Verb)
    * 표현(Representations)

2. REST의 특징
    * Uniform Interface - HATEOAS(Hypermedia As The Engine Of Application State)
    * Sateless - 상태값이 없음
    * Cacheable - HTTP 프로토콜 표준인 E-Tag 등을 이용한 캐쉬 가능
    * Self-Descriptiveness - Host, Content-Type 등 자신을 표현
    * Clinet-Server 구조
    * 계층형 구조 - L4, Proxy, GW를 통한 구조의 유연성
